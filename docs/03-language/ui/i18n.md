## 1. Filosofía

Kyle i18n es **compilado, no runtime-dynamic**. Las traducciones se resuelven en
tiempo de compilación y se empaquetan como datos estáticos. No hay archivos JSON
cargados dinámicamente — todo está en el binario.

| Aspecto | Enfoque tradicional | Kyle i18n |
|---------|:-------------------:|:---------:|
| Carga de traducciones | JSON/XLIFF en runtime | Datos compilados en WASM/binario |
| Resolución de locale | Runtime lookup | Switch estático por locale |
| Plurales | Librería ICU | Sistema tipado nativo |
| Formato fechas | `Intl.DateTimeFormat` | Tipos Kyle `DateFormat`, `NumberFormat` |
| Bundle size | Todo el diccionario | Solo el locale activo |

## 2. Declaración de Traducciones

### 2.1 Archivo de traducciones

Las traducciones se declaran en archivos `.ky` con sintaxis dedicada:

```kyle
# locales/es.ky — Español
locale "es":

    # Traducciones simples
    "app.title" = "Kyle Framework"
    "common.save" = "Guardar"
    "common.cancel" = "Cancelar"
    "common.delete" = "Eliminar"

    # Traducciones con parámetros
    "welcome.user" = "Bienvenido, {name}"
    "items.count" = "{count} elementos"

    # Plurales (reglas ICU)
    "items.plural" = (
        "=0"    => "No hay elementos"
        "=1"    => "1 elemento"
        "other" => "{count} elementos"
    )

    # Fechas
    "date.today" = "Hoy es {date, date, long}"
    "time.now" = "Son las {time, time, short}"
```

```kyle
# locales/en.ky — English
locale "en":

    "app.title" = "Kyle Framework"
    "common.save" = "Save"
    "common.cancel" = "Cancel"
    "common.delete" = "Delete"
    "welcome.user" = "Welcome, {name}"
    "items.count" = "{count} items"

    "items.plural" = (
        "=0"    => "No items"
        "=1"    => "1 item"
        "other" => "{count} items"
    )

    "date.today" = "Today is {date, date, long}"
    "time.now" = "It's {time, time, short}"
```

### 2.2 Locale registry

```kyle
# locales/registry.ky
from .es import es
from .en import en

registry LocaleRegistry:
    locales = {
        "es": es,
        "en": en,
    }
    default = "es"
```

## 3. Uso en Componentes

### 3.1 Texto traducido

```kyx
<text value=t("app.title") />

# Con parámetros
<text value=t("welcome.user", name: @user_name) />
```

### 3.2 Plurale

```kyx
<text value=t("items.plural", count: @items.len()) />
```

Según `count`:
- `0` → "No hay elementos"
- `1` → "1 elemento"
- `5` → "5 elementos"

### 3.3 Fechas y números

```kyle
# Formato automático según locale activo
fn format_birthday(date: Date) str:
    t("date.today", date: date)

fn format_price(amount: f64):
    # Usa formato del locale: $1,234.56 o 1.234,56 €
    number_format(amount, "currency")
```

### 3.4 Locale activo via Context

```kyx
@(
    context LocaleContext:
        locale: str = "es"
        t: fn(key: str, params: {str: str}?) str
        fn switch_locale(locale: str):
            locale = locale
            # recarga: los textos reactivos se actualizan
)

<view>
    <text value=t("app.title") />
    <button text=t("common.save") />
</view>
```

Componentes anidados acceden al mismo locale via `use_context(LocaleContext)`.

## 4. Sistema de Plurales

### 4.1 Reglas ICU

```kyle
locale "ru":
    "items.plural" = (
        "=0"   => "Нет элементов"
        "=1"   => "1 элемент"
        "one"  => "{count} элемент"     # 1, 21, 31...
        "few"  => "{count} элемента"    # 2-4, 22-24...
        "many" => "{count} элементов"   # 5-20, 25-30...
        "other"=> "{count} элемента"
    )
```

### 4.2 Categorías soportadas

| Categoría | Ejemplo (en) | Ejemplo (ru) |
|-----------|:------------:|:------------:|
| `=0`, `=1`, `=N` | Exact match | Exact match |
| `zero` | — | 0 |
| `one` | 1 | 1, 21, 31 |
| `two` | — | 2, 3, 4 (some langs) |
| `few` | — | 2-4, 22-24 |
| `many` | — | 5-20 |
| `other` | Everything else | Everything else |

### 4.3 Plural rules por locale

El compilador conoce las reglas CLDR de cada locale y genera el switch óptimo:

```javascript
// Generado automáticamente para locale "ru"
function pluralRule(count) {
    if (count === 0) return 'zero';
    if (count % 10 === 1 && count % 100 !== 11) return 'one';
    if (count % 10 >= 2 && count % 10 <= 4 && (count % 100 < 10 || count % 100 >= 20)) return 'few';
    return 'many';
}
```

## 5. Formato de Fechas y Números

### 5.1 DateFormat

```kyle
enum date_style:
    short     # 10/07/26
    medium    # 10 jul 2026
    long      # 10 de julio de 2026
    full      # viernes, 10 de julio de 2026

enum time_style:
    short     # 14:30
    medium    # 14:30:05
    long      # 14:30:05 GMT-3

final class date_format:
    locale: str
    date_style: date_style
    time_style: time_style?
```

### 5.2 NumberFormat

```kyle
enum number_style:
    decimal       # 1234.56
    currency      # $1,234.56
    percent       # 12.35%
    scientific    # 1.23E3

final class number_format:
    locale: str
    style: number_style
    currency: str?        # "USD", "EUR", "ARS"
    min_decimals: i32?
    max_decimals: i32?
```

### 5.3 Compilación

```javascript
// Generado automáticamente por locale
const formatters = {
    "es": {
        date: {
            short: { day: '2-digit', month: '2-digit', year: '2-digit' },
            long: { day: 'numeric', month: 'long', year: 'numeric' },
        },
        number: {
            decimal: { style: 'decimal' },
            currency: { style: 'currency', currency: 'USD' },
        }
    }
};
```

## 6. RTL (Right-to-Left)

### 6.1 Detección automática

```kyle
locale "ar":
    dir = "rtl"
    "app.title" = "إطار كايل"
```

### 6.2 Uso en componentes

```kyx
<view>
    @(dir = current_locale_dir())
    <vstack dir=@dir>
        <text value=t("app.title") />
        <button text=t("common.save") />
    </vstack>
</view>
```

El layout engine ajusta automáticamente:
- Flex direction invierte start/end
- Text alignment cambia a right
- Icons se reflejan horizontalmente (mirror)

## 7. Lazy Loading de Locales

Para apps grandes, los locales se cargan bajo demanda:

```kyx
@(
    fn load_locale(locale: str):
        # El runtime carga el locale del bundle
        load_locale_asset("locales/" + locale + ".wasm")
)
```

Por defecto solo se compila el locale `default`. Los demás se empaquetan como
bundle separado.

## 8. Compilación por Target

### 8.1 Web

```javascript
// ui-runtime.js — generado automáticamente
const translations = {
    es: { "app.title": "Kyle Framework", ... },
    en: { "app.title": "Kyle Framework", ... },
};

function t(key, params, locale) {
    let str = translations[locale][key] || key;
    for (const [k, v] of Object.entries(params || {})) {
        str = str.replace(`{${k}}`, v);
    }
    return str;
}
```

### 8.2 Desktop / Mobile

```kyle
# Generado automáticamente en el binario
fn t(key: str, params: {str: str}) str:
    locale = get_active_locale()
    match locale:
        "es":
            match key:
                "app.title" => "Kyle Framework"
                "welcome.user" => "Bienvenido, {name}"
                _ => key
        "en":
            match key:
                "app.title" => "Kyle Framework"
                "welcome.user" => "Welcome, {name}"
                _ => key

    # Reemplazar parámetros
    result = result.replace("{name}", params.get("name") ?? "")
    result
```

## 9. Buenas Prácticas

| Práctica | Descripción |
|----------|-------------|
| **Keys descriptivas** | `profile.edit.title` no `t23` |
| **Un solo archivo por locale** | `locales/es.ky`, `locales/en.ky` |
| **Locale registry** | Archivo central que lista todos los locales |
| **Fallback** | Si falta una key, mostrar la key misma o el locale default |
| **No concatenar** | `t("hello") + name` → usar `t("hello", name: name)` |
| **Traducciones en compilación** | El compilador valida que todas las keys existan en todos los locales |

---

## 10. Validación en Compilación

El compilador verifica:

1. **Keys faltantes**: Si `es.ky` tiene `"app.title"` pero `en.ky` no → **error**
2. **Parámetros faltantes**: Si la key requiere `{name}` y no se pasa → **error**
3. **Parámetros extra**: Si se pasan parámetros que la key no usa → **warning**
4. **Plural rules**: Valida que las categorías sean válidas para el locale

```bash
$ ky build app.kyx
❌ Error de i18n:
   - locale "en" falta: "profile.edit.title"
   - locale "es" key "welcome.user": parámetro {name} no se usa en la key
```
